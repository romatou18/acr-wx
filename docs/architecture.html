1 · System components (block diagram)

The serverless bridge: a 1-minute cron drives the weather-bot Lambda, which reads Gmail over IMAP, fetches weather, and posts replies back through Garmin. The weather-api Lambda serves the viewer/test endpoints. State lives in Turso.

2 · Message → reply (sequence diagram)

The end-to-end happy path for a single UPDATE, including the no-coordinates fallback (recover the fix from the message page) and the dry-run branch.

        Note right of B: shortlink, Lat/Lon, command

3 · Handler routing & routine broadcasts (flowchart)

What one handler() invocation decides: the pause gate, Garmin vs human-test sender routing, command handling, the delivery choke point, and the 07:00/19:00 NZ routine broadcast (which rebuilds a session from stored tokens — no fresh shortlink needed).

flowchart TD
  S["Cron tick → handler()"] --> PS{"paused_until > now?"}
  PS -->|yes| SKIP["Skip cycle
(mail stays UNSEEN)"]
  PS -->|no| POLL["IMAP poll UNSEEN"]
  POLL --> EACH{"new message?"}

  EACH -->|"no new mail"| RB{"07:00 / 19:00 NZ
window & not yet sent?"}
  RB -->|yes| BC["InitGarminSessionFromState
broadcast fresh report"]
  RB -->|no| DONE["done → flush debug_log"]

  EACH -->|yes| SENDER{"isGarminSender?"}
  SENDER -->|"human test"| TP["parseTestCommand
update · all · lat/long"]
  SENDER -->|Garmin| GP["parseGarminBody
shortlink · coords · command"]

  GP --> CMD{"command"}
  CMD -->|"START / STOP"| STATE["toggle active
in session_state"]
  CMD -->|UPDATE| FETCH["geofence → fetch → BuildReport"]
  TP --> FETCH

  FETCH --> DELIV{"GARMIN_DRY_RUN?"}
  DELIV -->|"1"| MAIL["sendToGarmin → email payload (SMTP)"]
  DELIV -->|"0"| POST["sendToGarmin → SendGarminReply
POST to explore.garmin.com"]
  POST --> OKCHK{"{Success:true}?"}
  OKCHK -->|yes| SAVE["saveState · logRequest · mark SEEN"]
  OKCHK -->|"no / error page"| RETRY["error logged;
message left UNSEEN for retry"]
  MAIL --> SAVE
  STATE --> SAVE
  BC --> SAVE

4 · Test loops (flowchart)

Three layers of coverage. Offline runs with no network or device (plain go test). Integration exercises the real email loop / live Garmin handshake. On-demand validates a freshly-captured page against the deployed parser. The committed golden page feeds the offline loop; /pause frees the cron so a fresh single-use page can be captured for the live/upload checks.
